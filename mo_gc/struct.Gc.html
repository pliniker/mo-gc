Non-atomic pointer type. This type is !Sync and thus is useful for presenting a Rust-ish API to a data structure where aliasing and mutability must follow the standard rules: there can be only one mutator.

Important note: even though this type is !Sync, any data structures that are composed of Gc pointers must still be designed with the awareness that the GC thread will call trace() at any point and so, must still be thread safe!

This is not a root pointer type. It should be used inside data structures to reference other GC-managed objects.
